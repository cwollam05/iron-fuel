For this assignment, I used Claude Web as my browser-based AI tool to build a dynamic menu filter feature for my Iron Fuel website. At first, the micro-iteration process honestly felt slow. Breaking everything into 5–10 minute steps seemed unnecessary because I usually prefer just building the whole feature at once. However, as I continued, I started to understand the value of this approach. Each small step forced me to actually review what was happening instead of blindly accepting whatever the AI generated.

The structured iteration made debugging much easier. Instead of having one large block of new code to troubleshoot, I only had to evaluate small changes at a time. This reduced the chance of introducing major errors and made the development process feel more controlled and intentional. When I used Claude’s self-review prompts, it mostly identified minor improvements rather than major bugs. The filtering logic worked correctly on the first implementation, but the self-review caught small refinements like improving button state handling and suggesting cleaner class toggling. Even though there weren’t serious errors, the self-review still added value by improving code clarity and structure.

One thing I noticed about using Claude Web is that it abstracts away some of the lower-level control compared to working directly with APIs. However, for front-end iteration and structured development, it was efficient and easy to use. Overall, micro-iteration felt slower at first, but I can see how it builds better habits for long-term development. It encourages intentional design, cleaner commits, and more reliable debugging patterns.
